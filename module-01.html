📌 Про сеть HQ-CLI (/28 и "не менее 16") В задании написано "не менее 16 адресов" — /28 даёт 14 хостов, что формально меньше 16. Однако рабочий гайд использует /28 и он работал на практике.
Возможная трактовка: 16 адресов в сети всего (14 хостов + сеть + broadcast = 16), что соответствует /28.
Используй /28 как в рабочем гайде. Если преподаватель укажет иное — меняй на /27.
📌 Сеть управления VLAN 999 (192.168.99.0/29) В задании упоминается сеть управления ≤8 адресов. Для этого используется 192.168.99.0/29 (6 хостов).
В упрощённой конфигурации (каждый сегмент на отдельном интерфейсе) эта сеть обычно не настраивается отдельно — её фиксируют только в отчёте.

3.3 Настройка SSH (порт 2026)

sed -i 's/#MaxAuthTries 6/MaxAuthTries 2/' /etc/ssh/sshd_config

⚠ Частая ошибка sed -i пишется СЛИТНО без пробела (не sed - i)!

9.2 Конфиг named.conf

HQ-SRV
sudo nano /etc/named.conf

Удали всё содержимое (Ctrl+K много раз) и впиши целиком:

options {
    listen-on port 53 { any; };
    listen-on-v6 port 53 { none; };
    directory "/var/named";
    allow-query { any; };
    recursion yes;
    forward first;
    forwarders { 77.88.8.7; 77.88.8.3; };
};

zone "sirius-exam.org" {
    type master;
    file "master/sirius-exam.org";
};

zone "100.168.192.in-addr.arpa" {
    type master;
    file "master/100.168.192.in-addr.arpa";
};

zone "200.168.192.in-addr.arpa" {
    type master;
    file "master/200.168.192.in-addr.arpa";
};

zone "255.168.192.in-addr.arpa" {
    type master;
    file "master/255.168.192.in-addr.arpa";
};

Сохрани: Ctrl+O → Enter → Ctrl+X

⚠ Частая ошибка! Пиши in-addr.arpa, НЕ ip-addr.arpa!

9.4 Прямая зона

HQ-SRV
sudo nano /var/named/master/sirius-exam.org
$TTL 1D
@       IN      SOA     hq-srv.sirius-exam.org. admin.sirius-exam.org. (
                        2025010101 1D 1H 3W 3H )
        IN      NS      hq-srv.sirius-exam.org.

hq-rtr  IN      A       192.168.100.1
hq-srv  IN      A       192.168.100.2
hq-cli  IN      A       192.168.200.2
br-rtr  IN      A       192.168.255.1
br-srv  IN      A       192.168.255.2
docker  IN      A       172.16.1.1
web     IN      A       172.16.2.1
📌 Почему docker → 172.16.2.1, а web → 172.16.1.1? nginx на ISP проксирует запросы: web.sirius-exam.org → веб-сервер HQ-SRV (через HQ-RTR), docker.sirius-exam.org → testapp на BR-SRV (через BR-RTR).
Клиент HQ-CLI достигает ISP через адрес 172.16.1.1 (ISP→HQ), а адрес 172.16.2.1 (ISP→BR) доступен через маршрутизацию.
Это соответствует /etc/hosts на HQ-CLI в задании 9 модуля 2. Если на экзамене nginx на ISP будет слушать только на одном IP — поставь оба на 172.16.1.1.

⚠
Частые ошибки и решения
❌ NAT на ISP слетел после перезагрузки
iptables правила не сохраняются — нужен rc.local
bash /etc/rc.d/rc.local
iptables -t nat -L -v -n   # проверяем что правила есть
❌ resolv.conf перезаписывается — браузер/dnf не работают
systemd-resolved ставит 127.0.0.53
rm -f /etc/resolv.conf
echo "nameserver 77.88.8.7" > /etc/resolv.conf
❌ SSH не стартует на порту 2026
SELinux блокирует нестандартный порт
semanage port -a -t ssh_port_t -p tcp 2026
systemctl restart sshd
# Если всё равно нет:
setenforce 0 && systemctl restart sshd
❌ OSPF не устанавливает соседство (show ip ospf neighbors — пусто)
Нет строки ip ospf в туннеле, или не выполнен clear
configure
tunnel gre 10
  ip ospf
exit
end
commit
confirm
clear ip ospf 1
❌ Eltex: "Syntax error" при вводе команды
Не в режиме configure
configure   # приглашение должно быть vesr(config)#
❌ HQ-CLI не получает DHCP-адрес
Забыли команду ip dhcp-server (включение сервера) на HQ-RTR
configure
ip dhcp-server
end
commit
confirm
❌ named не стартует
Ошибка в named.conf или правах файлов зон
sudo named-checkconf              # покажет строку с ошибкой
sudo chown -R root:named /var/named/master
sudo setenforce 0
sudo systemctl restart named
❌ sed -i не применяет изменения
sed -i пишется СЛИТНО, или строка уже была изменена
# Если не применилось — открой вручную:
nano /etc/ssh/sshd_config
